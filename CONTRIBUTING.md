# Contributing to Nexus

Thank you for your interest in contributing to Nexus!

## Philosophy

Nexus is intentionally minimal. We value clean, reliable collaboration between humans and agents over adding many features. Contributions that align with this philosophy are especially welcome.

## How to Contribute

- **Bug reports & feature requests**: Open an issue with a clear description.
- **Pull requests**: Fork the repo, make your changes, and open a PR. Please keep changes focused and well-documented.
- **Documentation**: Improvements to the README, AGENTS.md, or PROMPT.md are very appreciated.

## Development Setup

You can set up a local instance using the LLM-guided path (`PROMPT.md`) or follow the manual instructions in the README.

## Code Style

- Keep Docker and configuration changes minimal and well-commented.
- When modifying MCP abilities, update `AGENTS.md` accordingly.

## Questions?

Feel free to open an issue or start a discussion.
